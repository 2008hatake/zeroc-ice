Ice::UserExceptionFactory

Overview

local interface UserExceptionFactory

A factory for user exceptions. User exception factories are necessary in order to unmarshal an exception that is derived from an exception declared in the operation's throws clause. Ice does not support exception truncation, therefore if the actual type of the exception does not match those in the throws clause and a factory is not found, the client receives NoUserExceptionFactoryException.

Used By

Communicator::addUserExceptionFactory, Communicator::findUserExceptionFactory

createAndThrow

void createAndThrow(string type);

Create and throw a new user exception for a given user exception type. The type is the absolute Slice type name, i.e., the the name relative to the unnamed top-level Slice module. For example, the absolute Slice type name for a user exception Bar in the module Foo is ::Foo::Bar.

The leading "::" is required.

Parameters

type

The user exception type.

Called when the factory is removed from the communicator, or if the communicator is destroyed.
