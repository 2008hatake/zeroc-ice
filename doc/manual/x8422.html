Freeze::DBCursor

Overview

local interface DBCursor

A database cursor provides a way to iterate through all key/value pairs in the database. A cursor must be closed before its database is closed.

Used By

DB::getCursor, DB::getCursorAtKey, DB::getCursorAtKeyWithTxn, DB::getCursorWithTxn, clone

See Also

DB, DB::getCursor, DB::getCursorForKey

Operation Index

clone

Clone the cursor.

close

Close the cursor.

curr

Return the element to which the cursor currently refers.

del

Remove the key and the corresponding value from the database at the current cursor position.

getCommunicator

Get the communicator for this cursor.

next

Move the cursor to the next element in the database.

prev

Move the cursor to the previous element in the database.

set

Overwrite the data to which the cursor currently refers.

The cloned cursor.

Close the cursor. Subsequent calls to close have no effect.

    throws
	DBException;

k

Exceptions

If there are no further elements in the database.

DBException
