createEvictor

Evictor createEvictor(EvictorPersistenceMode mode)
    throws
	DBException;

Create a new evictor that uses this database to store identity/servant pairs.

Care must be taken to not to close this database, or the database environment this database belongs to, before the evictor has been properly deactivated. The evictor is deactivated by calling ::Ice::ObjectAdapter::deactivate on the object adapter in which this evictor is installed, or by shutting down all object adapters with ::Ice::Communicator::shutdown followed by ::Ice::Communicator::waitForShutdown.

Parameters

mode

The persistence mode for the new evictor.

Return Value

The new evictor.

Exceptions

DBException

Raised if the database has been closed.

See Also

Evictor, EvictorPersistenceMode

getCursorAtKeyWithTxn

DBCursor getCursorAtKeyWithTxn(Key k,
    DBTransaction txn)
    throws
	DBException;

Create a transactional cursor for this database. Calling curr on the cursor returns the key/value pair for the given key. The cursor operations will be transaction protected.

Care must be taken to not to close this database, or the database environment this database belongs to, before the Cursor has been properly closed.

The cursor must be closed before the transaction is commited or aborted.

Parameters

txn

The transaction context in which the cursor may be used.

k

The key under which the cursor is opened.

Return Value

A database cursor.

Exceptions

DBNotFoundException

If the key was not found in the database.

DBDeadlockException

Raised if a deadlock occurred.

DBException

Raised if any other database failure occurred.

See Also

sync

void sync()
    throws
	DBException;

Flush any cached information to the disk. Calling sync reduces the risk of losing data if the database is not closed properly.

If all the modifications to the database are done within the context of a transaction, it is not necessary to call sync. The transaction system ensures that all these modifications will be recoverable.
